    Setting Up The Shopify Google Shopping Tax Script

    Foreword

    This should only be used in accounts that are using the Shopify Google Shopping App. The purpose of this script is to override Merchant Center tax settings if they are porting over incorrectly from Shopify.

    Google Merchant Center Settings

    Create the tax and shipping settings within the Google Merchant Center:

    1. Copy tax settings from the Shopify store.

    Google Sheet Configuration

    The script should be implemented in the Google Drive of an account that has access to the to the client's Google Merchant Center. If the Merchant Center is in the MCA, place it in your drive. If that Merchant Center is linked to a dummy gmail account, the google sheet should be created in the dummy account's Google drive.

    1. Create new Google Sheet

      Title: LP-[Company]-[Purpose]

      Example: LP-ScratchWizards-Tax Script

    2. Modify Google Sheet Share settings to Anyone with a link can edit

    Script Configuration

    1. Within the Google Sheet, Navigate to Tools > Script Editor
    2. Paste Shopify Tax Script from shared Google Doc
    3. Insert Merchant and Account IDs > If this account is in the MCA set MerchantId to 112360638 and the subaccount as the accountId. Otherwise if the account is not in the MCA add the Merchant Center ID to both the merchantId and accountId field. > >Example (in MCA): >> >> Example (outside MCA):

    4. Add QuickBase order number to orderId

    5. List the purpose of your script. Example: 'Tax'

    Enabling Shopping Content API

    1. Within the Script Editor, Navigate to Resources > Advanced Google Services
    2. Name Project the same title as your Google Sheet
    3. Find Content API for Shopping

    4. Toggle On

    5. Enable Content API for Shopping in API console

    6. Enable Dashboar

    7. Search for Content API for Shopping > Enable

    Enabling & Authorizing Script:

    1. Return to Google Script

    2. Run getSettings and authorize script

    3. A pop-up will appear to authorize the application

    4. Click Review Permissions > Select account > Advanced > Go to [Project Name] (unsafe) > Allow

    5. Click Save

    6. Click Run ()

    Setting up Recurring Project Triggers:

    1. Select Current Project Triggers() and click +Add Trigger
    2. Choose which function to run: setTax
    3. Which runs at deployment: Head
    4. Select event source: Time-driven
    5. Select type of time based trigger: Day Timer
    6. Select time of day: Midnight to 1am >If using a dummy gmail a higher frequency would be sustainable. > >> Example: Time-Driven | Hour Timer | Every 4 hours
